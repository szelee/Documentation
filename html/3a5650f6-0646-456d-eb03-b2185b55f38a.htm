Remarks

BACKGROUND

This FEWS is intended to be used to request the season ticket fares which are available for a given journey and route. The input provided by the calling FEA supplies the origin and destination locations, the start date for the season ticket and, optionally, the period of validity. If the period of validity and ticket class is not provided then the CBE outputs a range of season tickets fares for standard class and, where available, first class.

This FIGS has also been updated as a result of SC CR102 to include fulfilment options in the output for the Web and Mobile channels for performance reasons, to avoid them calling FEWS 6.0. This is possible because the fulfilment options are more limited for these retail channels.

PROCESS DESCRIPTION

This FEWS is used to request the season ticket fares available for a given journey. This will be preliminary to season ticket purchase. If no period of validity (POV) is input then the following fares are returned:

  • Weekly

  • Monthly

  • 2, 3 and up to 11 Monthly

  • Annual

Where the period of validity is provided, only the fare for the designated period is returned by the CBE, fares may also include months and days eg 2 months 8 days (02M08D). If no Ticket Class is available, the response will include standard and first class fares.

There is a limit to how far in advance a season ticket may be purchased, however this fares request does not apply this limitation. (The advance season purchase limitation will be managed by the CBE within the appropriate purchase season ticket request web service call, in accordance with the latest RSP retail policies and Stagecoach retail requirements).

If Plusbus is available at the origin or destination location then this fare is also provided as part of the output. Where Plusbus is available and no POV is provided then the PlusBus fares returned are: 7 days, 1 month, 3 month and annual PlusBus season ticket fares. Where the POV is 7 days, 1 month, 3 month or annual the CBE returns the appropriate PlusBus season ticket fare. For other POVs no Plusbus fares are returned for the reason that PlusBus season ticket fares are only available for the defined periods of 7 days, 1 month, 3 month and 12 months.

Travelcard fares are not permitted as ‘stand alone’ add-ons to a season ticket. If travel in a London Underground zone is required as part of the ticket then it must be specified within the destination such that a single season ticket including the required London Transport zones can be retailed.

Multiple fares may be returned by the CBE in response to a season ticket fares request between two locations, where more than one fare exists. For example: a season ticket fares request for Manchester - Crewe will return more than one fare for each period of validity as there are different fares set by Northern Rail and by Virgin Trains. The Northern Rail season ticket fare is valid for use on any train operator’s service between Manchester and Crewe via any permitted route, whereas the Virgin Trains season tickets are subject to the restriction that they can be used on Virgin Trains services only. The different tickets have different route codes associated with them, and the restrictions on the Virgin Rail tickets are embedded in the route code.

This FEWS is for one passenger only, either adult or child. In some very limited cases a railcard discount may be applicable; this is applied by IPTIS where it is permitted (eg New Deal railcard). Normally railcard discounts do not apply to season tickets. The CBE will respond with those adult or child or discount fares as available in IPTIS. Where no ticket class is specified in the request, the CBE will respond with standard class and, where available, first class fares.

The output includes a flag to identify if a ticket fare returned is a season ticket base fare. This is provided so that any FEA using this can identify this fare from other fares and use it to decide whether or not this fare should be displayed to the customer (see NRES, which displays this information in its season ticket calculator). When printing a ticket the CBE will replace 7DS/7DF with the correct FTOT (PSS) to ensure the ticket displays SEASON <POV>.

It should be noted that if a railcard is specified in the input then the output from IPTIS will only be for Point to point full fares, which basically means the season ticket is not applicable for further railcard fare discount.

For On Board retail channel only 7 day season tickets are allowed, so only this fare should be returned. This response will be configurable by retail channel. In this version the FEWS output is amended to include the available fulfilment options from RCS for the web and mobile channels.

Fare Signature is a string that is returned within IPTIS Journey Planning response for each fare option. The fare signature contains the following information:

  • Ticket FTOT
  • Fare Setter
  • Route Code
  • Restriction Code
  • Fare Origin
  • Fare Destination
e.g. ‘SOS|IWC|00000|2C|0438|1072’

ASSUMPTIONS

  1. It is assumed the front end will manage the distinction between season tickets and non-season tickets, using this FEWS for season ticket fares requests and supplying the relevant data in accordance with the requirements of the interface.
  2. It is assumed that all season ticket types and fares calculation of these will be driven from the IPTIS fares feed, including managing customer selection of season tickets>10 months and 15 days, where all such fares are as for an annual 12 month season ticket.
  3. Travelcard fares will only be offered as inclusive fares and not as a separate Travelcard season ticket.
  4. It is assumed that the FEA will ignore the individual fulfilment costs returned or will interpret the results to ensure that costs are not doubled up on (e.g. A season ticket with Plus Bus via post will still cost £1.50 and not £3.00).

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Where the request message from the FEA fails one or more of the validation checks listed below, the CBE will respond with appropriate error message(s) as shown in the appropriate API guide.

  1. All mandatory data items are present.
  2. Start date must be in valid format.
  3. Period of validity must be in valid format.
  4. Origin and Destination locations are valid.
  5. Ticket Class to be Standard or First where present.
  6. Validate season ticket duration for all retail channels apart from On Board as follows.
    • Season ticket duration = 7day
    • Season ticket duration >= 1 Month and <= 1 year (12M00D).
  7. Start Date must be less than current date +x month (Configurable Booking Window Period).
  8. Railcard code is valid.
  9. In the case of a request from the onboard channel only 7 day season tickets are allowed so the PoV (period of validity) must be present and should be for 7 days.
