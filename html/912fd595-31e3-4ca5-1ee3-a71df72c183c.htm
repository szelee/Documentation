Remarks

BACKGROUND

The purpose of this FEWS is to allow the operator to request a reissue of ticket which has previously been non issued. This call to the CBE will then result in a print message request to Parkeon to reprint the ticket to the printer at the operator’s sales point.

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the ? this FEWS call occurs.

PROCESS DESCRIPTION

A reissue request typically happens where a printer has jammed and the tickets need to be reprinted. The reissue applies to the whole transaction and not for part elements, so all tickets in the basket will be re printed. This request is not used for reissuing tickets when a print failure of a walk up sale at a TVM has occurred.

A reissue of tickets can only happen where a non-issue of the original tickets has happened first and the corresponding non-issue BF/BN records have been written in SDCI+. In case the TVM non issues and refunds a ticket, the transaction is marked as non issue refunded and the CBE business validation checks this status when any reissue request is made at a ticket office and would reject such requests.

ASSUMPTIONS

  1. It is assumed that the trained operator knows that before a re issue request can be made the ticket must be first non-issued and cancelled.

VALIDATION

Parameter Validation:

  1. None

Business Validation:

  1. User credentials are valid and exist
  2. Transaction Number exists in CBE
  3. Check transaction’s Status = voided
  4. If transaction status is non-issue refunded then reject this request
