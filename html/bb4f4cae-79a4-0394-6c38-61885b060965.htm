Remarks

BACKGROUND

This FEWS allows the operator to cancel a reservation which is not required. The CBE then cancels the reservation in NRS. A reservation may be cancelled for a variety of reasons e.g. a refund of ticket with a reservation where the reservation has not expired.

PROCESS DIAGRAM

PROCESS DESCRIPTION

This call triggers a cancel reservation request to NRS by the CBE for all previously booked reservable items included in an NRS booking reference. The FEWS supports Call Centre and Ticket Office retail channels only and can be used in a variety of use cases not already covered by existing functionality.

A reservation should be cancelled if the ticket associated with the reservation has been refunded. If a ticket with a reservation is non-issued for any reason and is not being reissued then the reservations should be cancelled. The cancel reservation request should only be made if the reservation is still valid; i.e. the date of the reservation has not passed.

Only reservations that are associated with completed sales or refunds can be cancelled. Other reservations that are associated with pending sales states are cleared down automatically by the CBE under a separate job if the transaction is not completed.

The input includes the reservation reference number sales transaction number and the reason code for the cancellation. This code should be one of an enumerated list: refund, non-issue without reissue, not required and other. This call will cancel all reservations within this booking, i.e. if the reservation booking includes seats for 2 passengers it is not possible to part cancel one only. Ditto for passengers and bikes – the whole reservation with all elements will be cancelled.

On successful cancellation of the booking on NRS the CBE returns the date and time of the cancellation to the FEA. At The reservation cancellation date/time details are written to the sales transaction record for information so that it can be seen in FEWS 4.3.1 by sales support staff.

The cancellation of a reservation which relates to an advance purchase ticket (which is quota controlled with a mandatory reservation) is not permitted using this call. If a refund of this ticket is required then this is handled through the refund process (FEWS 9.n etc) which cancels associated reservations

ASSUMPTIONS

  1. It is assumed that a trained operator will know when to use this call to cancel reservations and that the associated FEAs makes this available as an option.
  2. It is assumed that the trained operator knows this call cancels all elements in a reservation. If some are still required then a new reservation will need to be made for the balance using FEWS 8.2 and 6.3.

VALIDATION

Parameter Validation:

  1. All mandatory items must be present.

Business Validation:

  1. User credentials must be valid if present.
  2. Reservation reference must exist and:
    • If transaction number is provided then the transaction must exist;
    • If transaction number is not provided, then the reservation being cancelled must not currently be associated with a sales transaction.
  3. The transaction state must be (or have been) confirmed.
  4. The reservation booking reference number exists within the transaction, where transaction exists.
  5. The cancellation of a reservation which relates to an advance purchase ticket (which is quota controlled with a mandatory reservation) is not permitted using this call. If a refund of this ticket is required then this is handled through the refund process (FEWS 9.n etc) which cancels associated reservations. This validation rule only applies where the reservation is already associated with a sales transaction.

Example of CancelReservation Request

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:ns="http://vixtechnology.com/cbe/2012-05-29">
  <soapenv:Header>
    <ns:SalesPointDetails>
      <ns:Nlc>9999</ns:Nlc>
      <ns:WindowNumber>1</ns:WindowNumber>
      <ns:MachineNumber>0001</ns:MachineNumber>
    </ns:SalesPointDetails>
  </soapenv:Header>
  <soapenv:Body>
    <ns:CancelReservationRequest>
      <!--Optional:-->
      <ns:ReservationBookingReference>BG648002</ns:ReservationBookingReference>
      <ns:ReservationCancellationReasonCode>Refund</ns:ReservationCancellationReasonCode>
      <!--Optional:-->
      <ns:SilverSeekRequestId>20</ns:SilverSeekRequestId>
    </ns:CancelReservationRequest>
  </soapenv:Body>
</soapenv:Envelope>
