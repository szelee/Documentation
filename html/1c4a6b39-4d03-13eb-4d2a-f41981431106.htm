GetCountryCodes
This call is part of the reference data and is available to all retail channels and does not require an input parameter. It is used to obtain the list of ISO 1366-1 alpha-2 country codes to enable country name to be available for screen display.
GetCouponDetails

The purpose of this FEWS is to provide the list of the coupon types which is not the default type (orange stock).

The coupon list is needed to support the FEA in advising the operator of the appropriate coupon types to be manually fed for printing items such as railcards, annual season tickets etc.

GetExtendedHorizonDetails
This FEWS is used by the FEA to provide extended dates, location pairs, excluded and restricted dates to enable bookings by customers in advance beyond the standard booking window (Up to 26 weeks). The FEWS will return extended dates that reveal prices based on applicable journey pattern and travel period. This will be used by FEAs to configure their journey planning calendar widgets to allow customer to select extended dates journeys.
GetFareGroupPermittedStations
This task requests fare group permitted stations. Request the stations that a customer can travel from or to based on the Fare Origin and Fare destination on their ticket.
GetFareRouteRestrictions
This call is part of reference data and is available to all retail channels. It is used to obtain route restriction codes used by TOCs to describe route restrictions in product design particularly promotions and other specific.The restriction data has additional details such as description, IDMS display name etc. The source data are from IPTIS reference data FareRouteRestrictions.
GetFaresLocationDetails
The purpose of this FEWS is to provide the fares location details. The source data for this is from IPTIS.
GetIncidents
This call returns incidents data and provides them to FEA. The source data is the NRES Incident XML feed. Only incidents relating to the retailing TOC are returned.
GetLoyaltySchemes
This case allows the FEA to fetch the various loyalty schemes registered in CBE.
GetNlcDetails
Returns the textual descriptions for each NLC, as per the fare locations data, to the Front End Application (FEA), in order that the FEA can interpret NLCs used in other FEWS and display the appropriate description information to the passenger.
GetNrsDetails
To provide reference data related to National Reservations System (NRS) such that the Front End Application (FEA) can interpret the codes used in reservation related FEWS and display the descriptive test to the passenger.
GetRailcards
This call is part of reference data and is available to all retail channels. It is used to obtain list of railcards with additional details. This call includes prices to purchase a railcard which is a feature of certain attended TIS only, (Ticket Office) for some but not all Railcards. Additionally, this call includes prices to purchase a railcard which is a feature of the ticket office only.
GetRangersAndRovers
Provides details and fares for all Ranger and Rover tickets.
GetRttiReasonCodes
This FEWS allows the front end to request RTTI Reason Codes to help to interpret Real Time Running data returned from the CBE.
GetSalesOfferings

This FEWS provides the Front end application with the Sales offerings that are available at the time they are loaded onto the Web server. Sales Offerings include the standard set of products, promotional products and attributes relating to a Sales Offering.

It is expected that this FEWS will be call at the beginning of the day by FEA and the response will be cached.

GetServiceProviders
This call is part of reference data and is available to all retail channels. It is used to obtain list of service providers i.e. Train Operating Company (TOC) with additional details. The source data are from IPTIS reference data Service Providers, with supplementary information provided by NRE.
GetSmartcardConfiguration
This FEWS is used by the FEA to provide smartcard configuration, including smart collection locations.
GetStationDetails
This task requests specific station information, (Facilities, opening times etc.) to be sent to the FEA, for a particular station using the CRS code from the Get Stations List (FEWS 10.2). The Station information is retrieved from two NRE XML feeds.It is supplemented by TOC specific stations data added by VTEC or SWTs Administration using the CBE Admin UI.
GetStationFareGroups
This case allows the FEA to fetch the station fare groups so that they know which stations are included in a group station fare.
GetStationGroups
The purpose of this FEWS is to provide the CRS codes for group stations to the FEA so that they can correctly display these. The source data for this is from IPTIS, which will be reviewed and approved by TOC Administrators as appropriate per TOC retail channel.
GetStationParkingDetails
The purpose of this FEWS is to allow an FEA to fetch station parking details for retailing. It is written so that it can also accommodate other parking products in the future e.g. bike storage. Only certain parking products can be retailed via a specific channel and the FEWS will filter out only relevant parking products for the stations based on the input channel.
GetStations

This call is part of reference data and is available to all retail channels.

Get Stations List provides the requester with a list of stations and a limited amount of associated data. The station details in this call includes the necessary basic information for use in Journey Planning and ticket purchased.

Additional station information, including TOC specific information can be provided to front end channels using FEWS 10.7 Get Station Information.

The source data for this FEWs is the IPTIS Stations reference data.

GetSupplements
This call is part of reference data and is available to all retail channels. It is used to obtain list of supplements with additional details. The source data are from IPTIS reference data SupplementDefinitions.
GetTicketFareGroups

This call is part of reference data and is available to all retail channels. It is used to obtain list of ticket fare groups.

Ticket Fare Groups are retail channel specific. They are used to organize the presentation of fares in the Retail Channel Front End applications. Fare Groups include a short and long description of the Terms and Conditions that will be applicable for all the Tickets Types in that group.

TOC Business Administration create and manage Ticket Fare Groups through Admin UI.

GetTicketRestrictions
This call is part of reference data and is available to all retail channels. It is used to obtain list of ticket restrictions with additional details. The source data are from IPTIS reference data TicketRestrictions, with supplementary information provided by NRE.
GetTicketTypes
This call is part of the reference data and is available to all retail channels. It is used to obtain list of ticket types with additional details such as description, display name etc. This data is provided from IPTIS with supplementary information provided by NRE.
GetTicketValidities
The purpose of this FEWS is to provide a look up for validity codes to the FEA. This data is provided by the CBE from IPTIS. This data is required by all retail channels.
GetTodLocations
This provides the FEA with all station locations (by NLC) where there are facilities for TOD collection at TVMs and booking offices (B O). The data should be used by the FEA to present to customers when making a TOD sale and is only required by those retail channels which support TOD sales.
GetTrainRunningTimes
Get train running times (using Live Departure Board – LDB). This is request for live updates of train running times of services as supplied by NRES Live Departure Boards(LDB). This request is by station location, for either arrivals or departures, for up to 2 hours forward.
GetUsageRules

This API returns the eVoucher applicable rules reference data.

This FEWS will return the Usage Rules for the eVouchers. The rules are used to apply applicability to the eVouchers when used as a method of payment in purchases.
