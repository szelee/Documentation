Remarks

BACKGROUND

The purpose of this FEWS is to allow the operator to confirm that the overprinting of the ticket coupons has been successful. This then triggers to creation of the non-issue record in SDCI+ by the CBE. For the On Board channel the call is used to trigger the creation of the SDCI non-issue record by the CBE.

PROCESS DESCRIPTION

This call is required by the CBE to trigger the creation of a non-issue record in SDCI, either a BF or a BN record. The call is required by the Ticket Office, Call Centre, and On Board retail channels to indicate that a non-issue has occurred and that original SDCI sale record in Lennon is to be reversed. The receipt of this call triggers the creation of the SDCI non-issue record either BF(for P2P tickets) or a BN(for season tickets).

For Ticket office and Call centre the call is the last step in cancelling a ticket that is required to be non-issued and confirms that the overprinting of the original ticket has occurred. This then triggers various updates in the CBE against the sale transaction fulfilment details and creates the non-issue records in the shift in SDCI. The input for these channels includes an overprint reason code which is held in the CBE, this indicates whether the overprint is due to a ticket non-issue, refund or replacement.

For the On Board channel where there is no overprinting of the ticket or this is outside the CBE’s control (for the On Board channel) this call is needed in conjunction with FEWS 9.7 to trigger the SDCI non-issue record creation by the CBE.

The output to this call is an ACK acknowledgement from the CBE.

ASSUMPTIONS

  1. The operator has been trained to confirm the successful overprinting of all coupons required to be cancelled.
  2. (RSPA2003 TIS Process Guide – Make a Sale) Where a sale and refund fall within the same day at the same location, the Non-Issue process may be used – the reason for non-issue is “wrong product”.

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Where the request message from the FEA fails one or more of the validation checks listed below, the CBE will respond with an appropriate error messages as shown in the appropriate API guide.

  1. User credentials are valid and exist
  2. Transaction Number must exist
  3. If NonIssue or Replacement OverprintType then for TransactionNumber can not exist lost in post paired transaction.
  4. If NonIssue or Replacement OverprintType then TOD is allowed for fulfilment only if the transaction is reissued transaction.
  5. If NonIssue or Replacement OverprintType then Transaction has to be confirmed.
  6. If overprintType Nonissue then NonIssueReason code has to be set for transaction.
  7. If overprintType Nonissue then Fulfilment status has to be confirmed.
  8. If overprintType Nonissue then Fulfilment status should not be failed
  9. If overprintType Nonissue then For Transaction can notshould exist P2P and season ticket which has reissued passanger fare.
  10. For overprintType Replacement can not be set NonIssueReasonCode to LostInPost.
  11. For overprintType Replacement must exist reissue ticket or railcard.
  12. If the overprint value is ‘refund’ then the transaction id must relate to a refund transaction id.
  13. If the overprint value is ‘refund’ then refund must be confirmed.
  14. If the overprint value is ‘refund’ then can not have confirmed overprint.
