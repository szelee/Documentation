Remarks

BACKGROUND

This FEWS returns details of all the coaches available on a train using the train service unique identifier (RSID) for a single journey leg from NRS. This call is available to all retail channels, except on-board, and only returns details of VTEC train coaches for the service, where the TIS operator is VTEC. This call has been updated as part of CR117.

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in reservation seat selection process this FEWS call occurs.

PROCESS DIAGRAM

Not required.

PROCESS DESCRIPTION

This FEWS is called by the FEA to get train coaches information for a specific train service and then map it to the coach’s visual layout as part of the seat picking process. The call returns details of all the coaches available on a train service using the unique RSID for a single journey leg from NRS. The call is typically made as the first part of the seat picking functionality available for VTEC train services. The steps for this are to get the coach details for the train service (FEWS 8.4), followed by get coach information (FEWS 8.5) for seats availability on a specific coach.

As a response, the FEWS returns details for each coach - the description and accommodation attributes which are available. The response to the call also includes the following error conditions which may arise from NRS and returns the appropriate error code and error message if they arise. Possible error messages from NRS as output include:

  • NRS SOPI Service Down – return #4.a
  • Invalid Origin CRS/Destination CRS/RSID/Journey Date – return #4.b
  • Train Coaches Details Not Available – return #4.c
  • Train Coaches Details Not Available – NRS Service Down.
  • Invalid Request Parameters.
  • Train Coach Details Not Available.

ASSUMPTIONS

  1. It is assumed that the FEA can translate the Coach Type codes and descriptions returned in the output by means of a mapping to be provided by VTEC.
  2. It is also assumed that the FEA shall maintain the coach visual layouts for the various coach types.
  3. It is assumed that when the customer opts to change seat using FEWS 8.6, the FEA will present a carousel layout of the various coaches for the customer journey.

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Failure to meet these checks will result in appropriate error messages as shown in the appropriate API guide.

  1. All mandatory fields are present.
  2. Origin & Destination CRS must be valid CRS.
  3. Journey Date must be a valid date and should not be before today.
  4. Appropriate error message to be returned if NRS SOPI Service is not available see section 3.2 above).
  5. User Credentials must be valid and exist (Ticket office and Call Centre only).
  6. Service Provide must be VTEC.
