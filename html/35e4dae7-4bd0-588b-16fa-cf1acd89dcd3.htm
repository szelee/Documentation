CbeWebServicePurchaseSmartcard Method

This call is used by web, mobile and call centre retail channels. It allows a request for the creation of a new or replacement smartcard to be initiated.

Puchase Smartcard creates the sale transaction record. The sale is then put into a pending state awaiting payment confirmation, (FEWS 6.2).

Remarks

BACKGROUND

This call allows a request for the creation of a new or replacement smartcard to be initiated. Validation is performed to ensure that all necessary parameters are correctly populated in the request, and that an admin charge is included where the customer has already received his allowed quota of free replacement cards.

Following a successful call to FEWS 6.12, front-end applications should then call FEWS 6.2 Purchase Confirmation to provide the payment details for any admin charge paid, otherwise a zero-amount cash payment.

Following Purchase Confirmation, CBE fulfilment will involve preparing and sending the card creation request to TMS, and handling the status callbacks received from TMS notifying the progress of the card creation.

PROCESS DIAGRAM

Not required.

PROCESS DESCRIPTION

Not required.

ASSUMPTIONS

  1. It is assumed that a customer can request a new smartcard (rather than a replacement) at any point, in order for example to update cardholder details. However any new cards will not be loaded with any products from an existing card.

VALIDATIONS

  1. All mandatory items must be present
  2. CustomerId must exist.
  3. If IsReplacementCard= true then IsrnToReplace is the ISRN for an active Smartcard currently linked to the customer account.
  4. DeliveryAddressId references an active address linked to the customer account.
  5. Correct admin fee is included in OtherCharges if a replacement card request exceeds the permitted number of free replacements, and WaiveAdminCharge is false.
