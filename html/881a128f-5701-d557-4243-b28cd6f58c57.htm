Remarks

BACKGROUND

This FEWS allows the FEA to request the best fare information from the CBE. The best fare is taken to mean the cheapest fare available for the journey requested. The CBE returns fares information for services to show only the cheapest fares as available in IPTIS. It provides the cheapest fares over a specific window period and route. Currently this window is limited by controls within IPTIS and NRS but in the future the configuration settings can be amended to allow a wider search. It differs from FEWS 2.1 in that no route or service information is shown and only the cheapest advance fares are provided. This FEWS is intended to be typically used by the FEA as a marketing feature to customers visiting the web site and is therefore restricted to the web and mobile retail channels only.

The call to the CBE will be used to obtain “Best Fares” for a designated journey, there will be no restrictions by the CBE on this but it may be limited by the FEA to a particular TOC services (likely to be the owning TOC).

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the business process this FEWS call occurs. This FEWS is a standalone request which can be invoked at any time without any previous information having been obtained by the FEA. Following a get best fare request the user may then request this fare using FEWS 2.1.

PROCESS DIAGRAM

PROCESS DESCRIPTION

This FEWS allows the user to request the best fares from the CBE using the IPTIS journey planning system, for a designated journey e.g. Manchester to London on a designated date/time. The input restricts the timeband to one of: All, Early Morning, Morning, Afternoon, Evening. These are set up in the CBE as pre-defined timebands. The start/end date should be limited to a single day, at the current time, by the FEA to avoid overloading IPTIS and NRS. When these systems can support greater timebands then this current restriction can be amended.

The volume of data provided from IPTIS depends on the loading on IPTIS. A request for ‘All’ timebands may not be met if the system is busy. This FEWS has configurable options to limit the response to CBE TOC as the Fare Setter. So if the CBE is set up for East Coast then the of Fare Setter value will be set to East Coast; only fares with East Coast as the fare setter in IPTIS will be provided back to FEA. (This is to restrict the best fare information to the CBE TOC operator).

The CBE response will always be for one adult, no railcard or promotional discounts are applied.

ASSUMPTIONS

  1. It is assumed that the FEA will restrict such requests for best fares for chosen journeys where the service is run by the owning TOC; the CBE will not limit this.
  2. It is assumed that the FEA may request several calls to make up a matrix of best fares over a particular time period for display to the customer for a designated set of routes.

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Failure to meet these checks will result in appropriate error messages as shown in the appropriate API guide.

  1. All mandatory fields are present.
  2. Station Locations are valid.
  3. All stations in input must be different locations.
  4. Start/End Dates < n weeks from current date (n= booking window currently 13 weeks).
  5. Travel date is not in the past.

Examples
Example GetBestFare Request
Copy
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:ns="http://vixtechnology.com/cbe/2012-05-29">
   <soapenv:Header>
      <ns:UserCredentials>
         <ns:UserName>TSawyer</ns:UserName>
         <ns:Password>password</ns:Password>
      </ns:UserCredentials>
    </soapenv:Header>
   <soapenv:Body>
      <ns:BestFaresInformationRequest>
         <ns:Origin>
            <!--Optional:-->
            <ns:Crs xsi:nil="true" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"/>
            <!--Optional:-->
            <ns:Nlc>0260</ns:Nlc>
         </ns:Origin>
         <ns:Destination>
            <!--Optional:-->
            <ns:Crs xsi:nil="true" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"/>
            <!--Optional:-->
            <ns:Nlc>1072</ns:Nlc>
         </ns:Destination>
         <ns:StartDate>2017-08-23T21:00:00</ns:StartDate>
         <!--Optional:-->
         <ns:EndDate>2017-09-17T10:00:00</ns:EndDate>
         <ns:Timeband>All</ns:Timeband>
      </ns:BestFaresInformationRequest>
   </soapenv:Body>
</soapenv:Envelope>
