ICbeWebServiceValidateCustomerPasswordResetToken Method

This FEWS covers the validation of an encrypted password reset token.

When the token is provided by the front end application, the CBE will recreate the encrypted token from internally held data and compare it with what has been provided. If the token has not expired and the comparison is successful, information related to the customer resetting their password will be returned to the FEA.

This is only required by web and mobile retail channels.

Remarks

BACKGROUND

This FEWS covers the validation of an encrypted password reset token by the CBE. This is only required by web and mobile retail channels.

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the process this FEWS call occurs.

PROCESS DESCRIPTION

This FEWS covers the validation of an encrypted password reset token which has been received via an email as a result of FEWS 4.5 request password reset. The CBE validates the token by recreating the encrypted token from internally held data and compares it with what has been provided. If the token has not expired and the comparison is successful, the information related to the customer resetting their password will be returned to the FEA.

This is only required by web and mobile retail channels.

ASSUMPTIONS

The output of security details and other fields are included for the FEA to enable them to do further validation as required/requested.

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Where the request message from the FEA fails one or more of the validation checks listed below, the CBE will respond with an appropriate error messages as shown in the appropriate API guide.

  1. Retail Channel is web or mobile.
  2. Validate Token using following method:
    1. Find token entry in table by using Token ID provided by FEA
    2. Check that expiry time has not passed
    3. Check that the token lifecycle status is Issued
    4. Concatenate GUID + “.” + Customer ID + “.” + E-Mail Address e.g. 184a1be9-4a9d-4e4b-8ae3-9e4869839395.12345.mjs@aol.com
    5. Do a SHA256 hash of string
    6. Compare with token provided by FEA
    7. Token is valid if the two hashes match
