Remarks

BACKGROUND

This FEWS will allow an operator at a ticket office to find a walk up sale made at a designated TVM or Ticket Office sales point location in a designated time period. This call is required to assist ticket office staff to deal with issues at a TVM or other sales point in the station, where the printer may fail in some way and the fulfilment cannot be completed.

PROCESS DESCRIPTION

  1. The purpose of this FEWS is to allow a ticket office operator to view the sales at a chosen TVM so that they can identify a walk up sale item which has been not been fulfilled due to printing or other issues. The FEWS also supports a ticket office operator to view sales made at other sales points in the ticket office if required.
  2. The output returns a list of sales transactions held in the CBE database in the last x minutes as per the minutes elapsed data item.
  3. From the output the operator can then identify the journey with the customer and find the transaction reference number. If the printer has failed/jammed then this most recent sale record output should be the one requiring to be non issued. The required data relating to the particular sale in the output can then be used to request a non issue and reissue using FEWS 9.7.
  4. The order of sales should be presented in most recent sold item first.
  5. This FEWS should not be used for failed TOD collection – this should be managed by FEWS 7.1 unlocking the CTR and reprinting it at the ticket office.

ASSUMPTIONS

  1. The minutes elapsed is left to the operator to set and is not validated, it is assumed that the operator is trained and does not allow this value to be excessive.
  2. It is assumed that the operator knows the sales point details for all TVMs and ticket office sales points in their station.

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Failure to meet these checks will result in appropriate error messages as shown in the appropriate API guide.

  1. All mandatory fields are present.
  2. All fields must be in correct format.
  3. The sales point details must exist in the list of TVMs table or Ticket office sales point table.
