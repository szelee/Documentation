ICbeReferenceDataServiceGetTicketValidities Method

The purpose of this FEWS is to provide a look up for validity codes to the FEA. This data is provided by the CBE from IPTIS. This data is required by all retail channels.

Namespace:  Vix.Cbe.WebService

[OperationContractAttribute]
[FaultContractAttribute(typeof(CbeFault))]
[FewsAttribute("10.18", FewsRevisionNumber.First, "1.0")]
Task<TicketValidityMessageResponseV1> GetTicketValidities()

Return Value

Type: TaskTicketValidityMessageResponseV1
Ticket validity response wrapped in a response message contract.
Remarks

BACKGROUND

It is envisaged that this data is requested by the FEA weekly.

This data is required by all retail channels.

All the data provided by IPTIS is included in the output but the FEA may only use a subset as required.

This is data is used by FEA to explain the ticket validities to the customer who is purchasing the ticket. The amount of information provided to the user depends on the FEA.

Break out and break return flags indicate whether break of journey is permitted on the leg.

BUSINESS PROCESS CONTEXT

Not required.

PROCESS DIAGRAM

None provided.

PROCESS DESCRIPTION

None provided.

ASSUMPTIONS

  1. It is assumed that the FEA will use some or all of this data to explain the ticket validities to the customer. The amount of data used will depend on retail channel and the design of the FEA.

VALIDATION

Parameter Validation:

None.

Business Validation:

None.
