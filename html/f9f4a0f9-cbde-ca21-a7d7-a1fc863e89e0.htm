Remarks

BACKGROUND

This FEWS is intended purely for test use, and provides a mechanism to introduce changes to reservation bookings that will have an impact on customer seat and bike reservations.

This will allow the controlled testing of all downstream activity including:

  1. Impact assessment performed by CBE to determine which customer reservation bookings are affected and whether they have been cancelled or changed.
  2. Messaging published to the TT Horizon SNS topic.
  3. Automatic updating of tickets for affected reservation itineraries.
  4. Additional information exposed via FEWS 4.2 Get Sales History and FEWS 4.3.1 Get Sales Transaction Details regarding the impact on affected transactions.
  5. Front-end application and 3rd-party handling of messaging and additional transaction information related to affected reservations.

BUSINESS PROCESS CONTEXT

PROCESS DIAGRAM

PROCESS DESCRIPTION

CBE will use the data provided in the request to populate a Reservation Alert Service (RAS) file and store it in S3 bucket in the same folder (TBC) as real RAS files generated by SilverRail.

For each Booking in the supplied request, CBE will need to ensure that an active Booking Header record exists containing the supplied BookingId.

When populating Segment Origin and Destination in the RAS file, only CRS values must be used.

When populating Segment DateOfTravel in the RAS file, only the date must be populated (see example in 3.2 above.

Once the RAS file has been placed in S3, CBE should initiate an execution of the Reservation Change Analysis Service (RCAS) if ExecuteRcas is set true.

ASSUMPTIONS

    VALIDATION

    Parameter Validation:

    1. All mandatory fields present.
    2. Parameters are correctly formatted.
    3. If there are no changes to reservations to a journey leg (eg it is not a service provided by LNER) a Segment must still be provided, and the Tickets collection or CancelledTickets collection must be populated with the existing reservation details (one Ticket per seat).
    4. If both the Tickets collection and CancelledTickets collection are provided then count of items must to be the same.

    Business Validation:

    1. BookingId must exist.
    2. Each CancelledTicket must correlate with existing booking leg. Or if CancelTickets is empty then each Ticket must correlate with existing booking leg.
    3. Each existing booking leg must correlate with provided CancelTickets or Tickets in request.
    4. Method should only be called when CBE is being used for testing.
    5. Carriage values for each ticket from Tickets list contain exactly one character.

    Copy
    <soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:ns="http://vixtechnology.com/cbe/2012-05-29">
  <soapenv:Header>
    <ns:SalesPointDetails>
      <ns:Nlc>9999</ns:Nlc>
      <ns:WindowNumber>1</ns:WindowNumber>
      <ns:MachineNumber>0001</ns:MachineNumber>
    </ns:SalesPointDetails>
  </soapenv:Header>
  <soapenv:Body>
    <ns:CreateReservationChangeRequest>
      <ns:Bookings>
        <!--Zero or more repetitions:-->
        <ns:Booking>
          <ns:BookingId>BS500001</ns:BookingId>
          <!--Optional:-->
          <ns:Segments>
            <!--Zero or more repetitions:-->
            <ns:Segment>
              <ns:DateOfTravel>2021-10-18</ns:DateOfTravel>
              <ns:Rsid>GR2200</ns:Rsid>
              <ns:Origin>
                <!--Optional:-->
                <ns:Crs>KGX</ns:Crs>
              </ns:Origin>
              <ns:Destination>
                <!--Optional:-->
                <ns:Crs>EDB</ns:Crs>
              </ns:Destination>
              <!--Optional:-->
              <ns:CancelledTickets>
                <!--Zero or more repetitions:-->
                <ns:Ticket>
                  <ns:TicketNumber>BS5000010-01</ns:TicketNumber>
                  <ns:Carriage>F</ns:Carriage>
                  <ns:SeatNumber>44</ns:SeatNumber>
                </ns:Ticket>
              </ns:CancelledTickets>
              <!--Optional:-->
              <ns:Tickets>
                <!--Zero or more repetitions:-->
                <ns:Ticket>
                  <ns:TicketNumber>BS5000010-02</ns:TicketNumber>
                  <ns:Carriage>F</ns:Carriage>
                  <ns:SeatNumber>16</ns:SeatNumber>
                </ns:Ticket>
              </ns:Tickets>
            </ns:Segment>
          </ns:Segments>
        </ns:Booking>
      </ns:Bookings>
      <!--Optional:-->
      <ns:ExecureRcas>false</ns:ExecureRcas>
    </ns:CreateReservationChangeRequest>
  </soapenv:Body>
</soapenv:Envelope>
