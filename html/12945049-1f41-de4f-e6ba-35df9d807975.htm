ICbeReferenceDataServiceGetStationGroups Method

The purpose of this FEWS is to provide the CRS codes for group stations to the FEA so that they can correctly display these. The source data for this is from IPTIS, which will be reviewed and approved by TOC Administrators as appropriate per TOC retail channel.

Remarks

BACKGROUND

The station id is used by the FEA to look up the details of the member stations (names and CRS codes) and plusbus details etc from FEWS 10.2.

This data is sourced from the Locations data (Admin review status = Accepted and the Group stations) from Iptis.

TOC Business shall define the group stations whitelist for each channel using the Admin UI.

BUSINESS PROCESS CONTEXT

Not required.

PROCESS DIAGRAM

None provided.

PROCESS DESCRIPTION

None provided.

ASSUMPTIONS

  1. This call assumes that FEWS 10.2 get Station details is called first by the FEA in order to look up the station details from the station id.
  2. There is a need for TOC Retail channels to consume only a subset of reference data as per their business requirements. For instance, TVM channel do not require ferry stations to be passed in reference data.
