Remarks

BACKGROUND

This is a call from the callcentre/refund operator to the CBE to request a refund application is rejected. This FEWS rejects the application following a validation failure response from FEWS 9.1.4 validate refund. This FEWS is currently only applicable to ETickets (Self-print and M-Tickets).

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the ticket purchase process this FEWS call occurs.

PROCESS DIAGRAM

Not required.

PROCESS DESCRIPTION

The purpose of this FEWS is to reject the refund application following a validation failure (FEWS 9.1.4). This FEWS is currently applicable to ETickets (Self-print and M-Tickets). Typically the refund of an eticket is rejected because the validtio0n chneck in FEWS 9.1.4 has shown the ticket to have been used and the refund assessment status value is 02 meaning not valid for a refund.

The input also allows the operator to add any notes to the refund record associated with this rejection. The reponse from the CBE is the refund reference number.

ASSUMPTIONS

  1. This FEWS will reject the whole refund application, which may have one or more tickets.
  2. This FEWS will only be invoked for E-Tickets.

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Failure to meet these checks will result in appropriate error messages as shown in the appropriate API guide.

  1. The refund number exists in the database
  2. The refund number is valid and refund state is In-progress
  3. The ticket in the refund is fulfilled by E-tickets.
  4. The RefundAssessmentStatus should be 02 (Ticket not valid for refund)
