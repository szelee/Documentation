Remarks

BACKGROUND

This call is part of reference data and is available to all retail channels.

It is used to obtain route restriction codes used by TOCs to describe route restrictions in product design particularly promotions and other specific. The restriction data has additional details such as description, IDMS display name etc.

The source data are from IPTIS reference data FareRouteRestrictions.

PROCESS DESCRIPTION

  1. An example entry would be code = 00000, Name = any permitted route.
  2. Industry Route Restrictions are loaded the CBE database and reviewed by TOC Administration such that this data may be filtered by channel. This is important for “on-Board” and “TVM” channels who will not need Route Restrictions created for promotions and other TOC specific products.
  3. To enable the display of customer text, the FEA is required to request data via the relevant reference web service via the CBEReferenceDataService.svc.
  4. Where user credentials are part of the input in the case of a TVM the username is a concatenation of sales point details namely: NLC, machine number and window number. The password is blank.
  5. The FEWS will return only those Route Restrictions that have ADMIN-REVIEW-STATUS set to “Accepted”. For more information, see technical note: Admin UI Group 1 Base Reference Data - Fare Routes & Supplements Technical Notes V1.1.

ASSUMPTIONS

  1. It is assumed FEA will keep fare route restrictions list in a cache to lookup route codes used in journey planning calls responses.
  2. It is responsibility of the FEA to keep a cache up-to-date i.e. by synchronizing the data on daily basis.

VALIDATION

None
