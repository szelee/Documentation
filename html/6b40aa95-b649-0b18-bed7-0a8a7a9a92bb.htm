Remarks

BACKGROUND

This call allowing to redeem one day travel pass for flexi season ticket in his 28day period of validity based on parameters specified in the request:

  • Customer id
  • Flexi-Season transaction number against which the travel pass is being redeemed
  • Required travel datetime
  • Transaction numbers for any existing reservations that need to be paired with the travel pass

BUSINESS PROCESS CONTEXT

PROCESS DIAGRAM

PROCESS DESCRIPTION

Customers will be able to ‘redeem’ one of their 8 travel passes by following a booking flow that allows them to make seat (and bike) reservations on specific outbound and return journeys on the desired day of travel.

The travel pass will be fulfilled as a season eTicket with a 1-day period of validity. All associated seat and bike reservations will be made as standalone reservation purchases and linked to the travel pass via transaction pairing.

Each redeemed travel pass will be associated with the relevant Flexi-Season product (again via transaction pairing), thereby decrementing the number of remaining passes available.

Each pass will be assigned one of the 8 UTN values created when the Flexi-Season was purchased.

Once all 8 passes have been redeemed or the product has exceeded its period of validity, then it will no longer be possible to redeem further passes.

Each Flexi-Season is for a single passenger, and the travel passes that can be redeemed using it are non-transferable.

Passenger name will be displayed on each travel pass, together with photocard id.

ASSUMPTIONS

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Where the request message from the FEA fails one or more of the validation checks listed below, the CBE will respond with an appropriate error messages as shown in the appropriate API guide.

Parameter Validation:

  1. All mandatory items present.

Business Validation:

  1. Customer must exist
  2. Transaction number must exist in CBE.
  3. Transaction number must belong to customer.
  4. Transaction has been refunded or had a refund request made.
  5. The required date of travel is within the Flexi-Season’s period of validity.
  6. The travel pass has already been redeemed for the current day from the same Flexi-Season.
  7. The Flexi-Season still has at least one unredeemed travel pass available.
  8. Reservation does not exist.
  9. The booking reference does not belong to customer identifier.
  10. Invalid reservation status.
  11. The reservation is not for date of travel.
  12. The required date of travel is for the current day.
