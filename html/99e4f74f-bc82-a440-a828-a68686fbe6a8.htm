Remarks

BACKGROUND

This FEWS allows the user to access their customer account details held in the CBE database. The current customer account information, held by the CBE, is supplied to the FEA on receipt of a valid customer login request (FEWS 4.4) as shown below).

This call has been updated to return the revised customer account details following CR95 (Loyalty) and CR99 (Customer account enhancement).

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the business process this FEWS call occurs. This FEWS is a standalone request which can be invoked at any time without any previous information having been obtained by the FEA.

PROCESS DESCRIPTION

This web service allows the customer to retrieve their account details that are held in the CBE database. This web service call is made following a successful customer logon web service call; the response to the successful customer logon supplies the full customer record to the FEA. The FEA will then allow the customer to update the record details as required using FEWS 4.1.

The output depends on the type of customer account, with partial account details containing the least, to the most details found in (full) customer accounts with guest accounts included in between. The output returns all possible attributes associated with a full customer account.

ASSUMPTIONS

It is assumed that the FEA will only display and use those items as required.

It is assumed that the customer can specify one or more values for each Contact Preference(Booking Enquiry, Special Offers, Other).

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Failure to meet these checks will result in appropriate error messages as shown in the appropriate API guide.

  1. Customer Id must exist.
