Remarks

BACKGROUND

This FEWS allows the TVM FEA to confirm to the CBE that a refund of payment has been made following a walk up sale which has then been non issued (FEWS 7.6).

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the TVM non issue process this FEWS call occurs.

PROCESS DIAGRAM

PROCESS DESCRIPTION

This call follows FEWS 7.6 TVM request cancellation. On receipt of this FEWS 7.7 the CBE will update the original sales transaction status to void TVM refunded, together with the date and time of change. This call can be seen as a completion of the TVM non issue and refund process and provides feedback to the CBE of this.

The CBE then creates a mini refund record against the sale transaction, this is only created when the sale transaction state moves from void (non issued, following FEWS 7.6) to void TVM refunded.

If the sale transaction is not found by the CBE as it has already been cleared down (in the case where no FEWS 6.2 request has been received), then the CBE will return an error message no refund record is created.

ASSUMPTIONS

  1. The TVM FEA will manage the refund of money by card or cash depending on the original MOP of the sale.
  2. If the CBE ignores the call and receives an error message - transaction not found or cancelled the TVM will not need to follow up on the request.
  3. This call does not apply to TOD/CTR requests and should not be used for this.

VALIDATIONS

  1. All mandatory parameters are present
  2. Sales point details exist and are valid
  3. Transaction number exists and is in state non issued
  4. Refund amount matches the transaction value.
