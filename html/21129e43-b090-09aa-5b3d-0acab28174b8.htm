Remarks

BACKGROUND

This FEWS provides a Sales offering Access authorisation (or “door keeper”) service for Promotions.

The FEWS validates the Access information required for the Sales Offering, (Pin Number, email address, email Domain etc.) and then checks the availability (effective dates and Quotas) of the Sales Offering. The response will accept or reject the request for authorisation.

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the business process this FEWS call occurs. This FEWS is a standalone request which can be invoked at any time without any previous information having been obtained by the FEA.

PROCESS DESCRIPTION

The Sales Offering detail will include the method used to access that Sales Offering. This may be a pin number, email or be open to anyone. This is validated against an Access Control list.

Additionally, a Sales Offering (i.e. promotions) may have a set of quotas. These have to be checked before access is allowed to the sales offering. The default value of the quotas is null.

ASSUMPTIONS

  1. It is expected that the FEA will be responsible for the customer journey based on the outcome of this request. If accepted, the FEA can use the Sales offering Code and Access Control Id with subsequent FEWS.
  2. If access is not granted, the FEA will be responsible for communicating to the customer.
  3. Other FEWS will not be expected to validate access to the Sales offering.

VALIDATION

The following validation checks are made by the CBE on the data provided by the FEA. Failure to meet these checks will result in appropriate error messages as shown in the appropriate API guide.

  1. The PRM_ENABLE_SALES_OFFERING (Config UI) must be set to TRUE.
  2. The sales offering Code is valid and current.
