Remarks

BACKGROUND

This FEWS returns all available seats for a maximum of 3 coaches for a train (RSID) associated with a journey leg. The FEWS triggers a request from the CBE to NRS and then returns the NRS response for available seats as output. This FEWS This call is available to web and mobile and telesales retail channels and only returns details of VTEC train coaches for the service, where the TIS operator is VTEC.

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the reservation process this FEWS call occurs.

PROCESS DIAGRAM

Not required

PROCESS DESCRIPTION

This call is used in conjunction with FEWS 8.4 (get coaches information) to provide a seat picking facility to the user. The call returns only the available seats (not all seats), in up to 3 coaches on a train service as identified by the unique RSID and date and time. In addition to the seat availability details the seat attributes for each seat is returned, these relate to the position in the carriage e.g. by the aisle, by a luggage rack etc.

The FEA will consume the response from this FEWS and map available seats on the train coaches in the coach visual layout, in order for the user to select a preferred seat as part of the reservation booking process. This data is then used when making a reservation booking.

The coach id(s) provided in the input can be identified by a 4 digit unique identifier that is returned for each coach for the train in FEWS 8.4 response, and need to be passed in this FEWS input to return seat availability details for the required coach.

Where the NRS service is unavailable the appropriate error message from NRS is passed back as a response to this call.

In this version as part of CR117 getting seat picking details is limited to VTEC services only so the service provider field has been added to the input to check.

ASSUMPTIONS

  1. It is assumed that the FEA knows the list of codes that are required to be used in the input and translates these for the user.
  2. It is assumed that the FEA will build the coach seat layout separately with Stage Coach and map the available seat data returned in the response to the coach seat layout [diagrammatic display of availability].

VALIDATION

Parameter Validation:

  1. None

Business Validation:

  1. All mandatory fields must be present.
  2. Journey Date must be a valid date and should not be before today.
  3. Appropriate error message to be returned if NRS SOPI Service is not available.
  4. Coach id must exist in NRS for the RSID
  5. If accommodation unit type is present the value must be valid
  6. User Credentials must be valid and exist (retail channels Call Centre and Ticket Office)Service Provide must be VTEC
  7. Check using NRS configuration that this NRS message type has been disabled for this channel
