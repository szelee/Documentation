Remarks

BACKGROUND

This FEWS is to allow a TVM to advise the CBE that print job at a TVM printer has been completed, so that the fulfilment can be recorded as completed by the CBE. (This is part of the Fetched delivery solution CR058 for Parkeon TVMs). The call is only applicable to the TVM retail channel.

BUSINESS PROCESS CONTEXT

The diagram below shows the context of where in the where in the TVM fulfilment this FEWS call occurs.

PROCESS DESCRIPTION

This FEWS is used by the TVM to inform the CBE that either a walk up sale has been successfully printed, or a CTR has been successfully retrieved and printed. On receipt of this call, where the call refers to a CTR collection this triggers the following tasks by the CBE: SDCI creation of the TOD collection and sale elementThe update of the CTR journey record print/booking status flag to 4/3 on the LSM data base, for TOD ticket collection at the TVM

Where the call refers to a walk up sale, the input will be the sales transaction number and on receipt of this the CBE will update the sales transaction record with the fulfilment print status as completed.

For walk up sales the call includes a field in the input to identify,) if the sales receipt has been printed by the TVM; a receipt coupon is always supplied but the customer may not want it so the TVM will not print it. This field is required so that the number of coupons in the input can be correctly validated by the CBE. If the number of coupons is incorrect then an error message is triggered by the CBE.

The should TVM will not send FEWS 7.5 messages where the coupon print is incomplete or part failed (except in error); in these cases it will send instead FEWS 7.6. If an incorrect FEWS 7.5 message is received then the CBE will respond with an error message and will not alter the print request state indicator. (See TVM Fetched delivery Doc V4.3 for details of states).

Assuming that the print message supplied as the output of FEWS 7.2 has been successfully printed by the TVM, the TVM FEA will now send a print completed message (FEWS 7.5). On receipt of this FEWS call where a CTR detail is provided the CBE updates the status of the booking/print flag to 4/3 in the CTR record and writes this to the LSM database for the journey(s) requested. At this point the CBE releases the lock on the CTR record on the LSM database. Once the CTR record has been updated on the LSM database the CTR record is deleted from the CBE cache.

Note that there is no change to the input and output data as a result of these amendments to the CBE.

ASSUMPTIONS

  1. This assumes that the CBE has previously provided the fetched print message on completion of the direct sale CBE sale transaction (fetched delivery) or a CTR print request. See FEWS 7.2.
  2. It is assumed that if the receipt coupon was not required by the customer then the TVM will not issue it, so the BV check will confirm that it was not printed.

VALIDATION

  1. All mandatory fields are present
  2. Only one of the following mutually exclusive parameters is allowed in request: Ctr Reference | transaction number.
  3. The sales point details are valid within the CBE set up. (Combination of NLC, window number, machine number)
  4. The sale transaction should exist and be in state confirmed or completed (following FEWS 6.2)
  5. If the IsSalesReceiptPrinted flag is Y then the number of coupon print details should match the number of coupons provided in the original print message
  6. If the IsSalesReceiptPrinted flag is N then the number of coupon print details should be one less than the number of coupons provided in the original print message
  7. Print status should be one of list (see below)
  8. The CTR record and journey reference details must exist in the local cache of CTR records
  9. The booking/print flag for the CTR record in the local cache must be either 9/2 or P/2

Example PrintCompleted Request
Copy
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:ns="http://vixtechnology.com/cbe/2012-05-29">
   <soapenv:Header>
      <ns:SalesPointDetails>
         <ns:Nlc>1072</ns:Nlc>
         <ns:WindowNumber>1</ns:WindowNumber>
         <ns:MachineNumber>0001</ns:MachineNumber>
      </ns:SalesPointDetails>
   </soapenv:Header>
   <soapenv:Body>
      <ns:PrintCompletedRequest>
         <!--Optional:-->
         <!--Optional:-->
         <ns:SalesTransactionPrintDetails>
            <ns:TransactionNumber>17SMP09VIJ</ns:TransactionNumber>
            <ns:CouponPrintDetails>
               <ns:CouponPrintDetail>
                  <ns:CouponNumberId>1</ns:CouponNumberId>
                  <ns:PrintStatus>Complete</ns:PrintStatus>
               </ns:CouponPrintDetail>
               <ns:CouponPrintDetail>
                  <ns:CouponNumberId>2</ns:CouponNumberId>
                  <ns:PrintStatus>Complete</ns:PrintStatus>
               </ns:CouponPrintDetail>
               <ns:CouponPrintDetail>
                  <ns:CouponNumberId>3</ns:CouponNumberId>
                  <ns:PrintStatus>Complete</ns:PrintStatus>
               </ns:CouponPrintDetail>
            </ns:CouponPrintDetails>
            <ns:NumberOfCoupons>3</ns:NumberOfCoupons>
            <ns:IsSalesReceiptPrinted>true</ns:IsSalesReceiptPrinted>
         </ns:SalesTransactionPrintDetails>
      </ns:PrintCompletedRequest>
   </soapenv:Body>
</soapenv:Envelope>
